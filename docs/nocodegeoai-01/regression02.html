실습 6 - K-NN

데이터 만들기

  1. “Paint Data” 위젯을 추가하고 더블클릭한다.

  2. 마우스를 이용해 이런 모양으로 데이터를 그린다.

X축의 값Y축의 값
0 ~ 0.20.2
0.2 ~ 0.40.4
0.4 ~ 0.60.6
0.6 ~ 0.80.8
0.8 ~ 1.01.0
  1. 만들어진 데이터의 갯수는 여기서 확인할 수 있다. 여기서는 마우스를 그림을 그려 총 456개의 데이터를 만들어 냈음을 알 수 있다.

데이터의 Feature와 Target 구분하기

Paint Data는 데이터를 쉽게 만들어 주지만 target을 지정해 주지 못한다. 그래서 Select Columns 위젯을 이용해 target을 지정해 준다.

  • Feature: 독립변수, X, target 값을 결정하는 요인.

  • Target: 종속변수, Y, 주택가격 등 feature 들에 의해 결정된 값.


  1. Transform에서 “Select Columns” 위젯을 추가한다.

  2. Select Columns 창이 나타나면 Features 항목에서 “Y” 를 Target 항목으로 끌어내린다.(드래그 앤 드롭)

  3. 닫기 버튼을 누른다.

이제 X축의 값이 feature이고, Y축의 값이 target이 되었다.


K-NN 성능 테스트

  1. Model에서 “KNN” 위젯을 추가한다.

  2. Evaluate 에서 “Test and Score” 위젯을 추가한다.

  3. “KNN” 노드와 “Test and Score” 노드를 연결한다. 연결선에 “Learner” 라고 적혀 있어야 한다.

  4. “Select Columns” 노드와 Test and Score“ 노드를 연결한다. 연결선에 “Data” 라고 적혀 있어야 한다.


  1. “Test and Score” 노드를 더블클릭한다.

  2. RMSE값을 보니 충분히 훌륭한 결과를 보여준다.

  3. R2값을 보니 충분히 훌륭한 결과를 보여준다.


K-NN 동작 테스트

특이하게 입력한 데이터를 이용해 K-NN의 동작을 테스트 해 보자.

  1. Transform에서 “Create Instance” 위젯을 추가한다.

  2. “Select Columns” 노드와 “Create Instance” 노드를 연결한다.

  3. 연결하자 마자 “Edit Links” 창이 나타나는데 “Data - Data“로 되어 있는 것을 확인하고 “OK” 버튼을 누른다.


  1. Evaluate에서 “Predictions” 위젯을 추가한다.

  2. “KNN” 노드와 “Predictions” 노드를 연결한다. 연결 선에 “Model - Predictions” 라고 나타나야 한다.

  3. “Create Instance” 노드와 “Predictions” 노드를 연결한다. 연결 선에 “Data” 라고 나타나야 한다.


  1. “Predictions” 노드를 더블클릭한다.

  2. Predictions 창이 나타난다.

  3. “Create Instance” 노드를 더블클릭한다.

  4. Create Instance 창이 나타난다. 이 때 Predictions 창이 뒤로 가려서 안보일텐데, 모니터 화면 맨 아래 태스크바에서 “Predictions“를 누르면 다시 나타나서 위 그림과 같은 상태가 된다.

  5. 독립변수인 x의 값을 슬라이드를 조절해 변경한다.

  6. x값이 바뀔 때마다 KNN에서 찾아낸 값이 달라지는 것을 확인할 수 있다.

Error의 값은 신경쓰지 말자. 우리가 Create Instance 창에서 x값만 바꾸고 정답인 y값은 바꾸지 않았기 때문이다.


테스트할 요소들

기본적으로 K(확인할 이웃의 수)의 값은 5이다. 설정을 만지지 않았다면 K=5인 상태다.

독립변수 X

독립변수인 x의 값을 0.1, 0.3, 0.5, 0.7. 0.9로 순서대로 바꿔보자.

그 값들은 가로로 길게 준 값들의 중간쯤이다.


x = 0.1인 경우,

y = 0.2로 동일하게 준 그 가로선의 중간쯤이 되고, 이 부근에서 x축 상에서 가까운 이웃을 찾으니 빨간 원안에 있는 데이터가 될 테고 모두 y=0.2라서 K-NN의 결과도 아래와 같이 0.2근처의 값이 나타난다.


x = 0.3인 경우,

y = 0.4로 동일하게 준 그 가로선의 중간쯤이 되고, 이 부근에서 x축 상에서 가까운 이웃을 찾으니 빨간 원안에 있는 데이터가 될 테고 모두 y=0.4라서 K-NN의 결과도 아래와 같이 0.4근처의 값이 나타난다.


K(계산에 사용할 이웃의 수) 변경

지금까지는 기본값인 K=5인 상태에서의 테스트였다.

K=100으로 바꾸고 테스트해 보자.


아까와 같이 x=0.112인 경우

똑같이 x=0.112로 주었는데,

  • 아까 k = 5일 때는 0.203581,

  • 지금 k = 100일 때는 0.266492 이다.

이유는,

100개의 이웃을 찾아서 평균을 내야 하는데, 아래에는 대략 90개의 자료 밖에 없어 x축에서 가장 가까운 0.25 정도까지의 이웃까지 찾아서 그 값을 평균냈기 때문이다.

X값이 0.2 ~ 0.4까지의 Y값은 모두 0.4이기 때문에 평균이 약간 올라갔다.
