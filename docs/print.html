실습 1 - iris

실습 설명: Iris(붓꽃) 분류

Iris(붓꽃)의 품종은 백가지 종류가 넘지만 여기서는 딱 3개만 구분해 보자.

  • setosa(세토사)“ 꽃잎이 상대적으로 매우 작고 둥근 편.
  • versicolor(버시컬러): 중간 크기, 푸른색과 보라색이 강하다.
  • virginica(버지니카): 꽃잎과 꽃받침이 가장 크고 길쭉하다.

사진 출처: Iris Flower Classification - Peace Ikeoluwa Adegbite 미디엄

구분하는 기준은 다음과 같다.

  • 꽃받침(sepal)의 길이
  • 꽃받침(sepal)의 너비
  • 꽃잎(petal)의 길이
  • 꽃잎(petal)의 너비

길이, 너비 데이터의 숫자 단위는 모두 cm 이다.


파일 읽어오기

  1. 위젯 중 “File“을 클릭한다.

  2. 추가된 “File” 노드를 더블클릭한다.

  3. 파일 선택 창이 나타난다.

  4. Source에서 “iris.tab“을 선택한다.

  5. 창을 닫는다.


필드명을 한글로 바꾸기

생소한 영어단어들 때문에 학습에 방해가 되니, 필드명을 한글로 바꾸자.

다음과 같이 바꾼다.

  • sepal length: 꽃받침 길이
  • sepal width: 꽃받침 너비
  • petal length: 꽃잎 길이
  • petal width: 꽃잎 너비
  • iris: 품종

  1. 왼쪽 위젯 중 “Edit Domain“을 선택한다.

  2. “Edit Domain” 노드가 나타난다.

  3. “File” 노드와 “Edit Domain” 노드를 연결하고 “Edit Domain” 노드를 더블클릭한다.

  4. “Edit Domain” 창이 나타난다.

  5. Iris.tab 이 가지고 있는 데이터의 필드 목록이 나타난다. 이 중 제일 위 “sepal length“를 선택한다.

  6. “꽃받침 길이“라고 입력한다.

  7. “Apply” 버튼을 누른다. “sepal width → 꽃받침 길이” 라고 나타난 걸 볼 수 있다.


위와 같은 방식으로 5개 필드 모두 바꾸고 창을 닫는다.

버지니카의 꽃잎은 80% 정도는 5cm 정도이고, 20% 정도는 7.5cm 정도임을 알 수 있다.


box plot(상자 수염 그림)

box plot은 다음과 같은 통계적 특성을 한 번에 보여준다.

  • 최대값
  • 최소값
  • 중앙값(median. 평균값이 아닌 크기 순으로 정렬했을 때 가장 중앙에 있는 값)
  • 사분위수(크기 순으로 정렬했을 때 하위 25%, 하위 50%, 하위 75%의 값)

  2. “Data Table“과 “Box Plot“을 연결한다.

  3. “Box Plot“을 더블클릭한다.

  4. Box Plot 창이 나타난다.


box plot의 의미

위 그래프에서 제일 오른쪽에 있는 버지니카를 보면서 box plot 각 항목의 의미를 확인해 보자.

표준편차 : 0.5463


1사분위 : 5.1


2사분위 = 노란색 세로선 = 중앙값 = 제일 가운데 값(median) : 5.55

전체 데이터 영역

점선으로 표시한 영역이 전체 데이터 영역


실선으로 표시한 부분의 왼쪽 끝 : 평균 - 표준편차 = 5.552 - 0.5463 인 지점


실선으로 표시한 부분의 오른쪽 끝 : 평균 + 표준편차 = 5.552 + 0.5463 인 지점

AI 학습

그래프 등을 통해 여기까지 본 사람은 이제 대체적으로 ‘아주 작은 꽃은 세토사’ 라는 등의 특징을 알 수 있지만 AI는 알지 못한다. AI에게 이런 특징을 학습시키자.

  1. 기존 시각화 노드들은 모두 지우고 시작한다.(놔둬도 상관없지만 불편하다)

  2. Model 위젯 중 “KNN” 위젯을 클릭한다.

  3. Evaluate 위젯 중 “Test and Score” 위젯을 클릭한다.

  4. “Data Table” 노드와 “KNN” 노드를 연결한다.

  5. “Data Table” 노드와 “Test and Score” 노드를 연결한다.

  6. “KNN” 노드와 “Test and Score” 노드를 연결한다.

  7. “Test and Score” 노드를 더블클릭한다.

  8. 학습 결과를 확인한다.


KNN 위젯

  • KNN 알고리즘에 의해 동작하는 AI 모델.

  • K-NN(K-Nearest Neightbors, K-최근접 이웃) 위젯

  • 학습한 결과가 여기에 저장된다 = 가중치가 여기에 저장된다.

  • 이제 붓꽃의 꽃잎, 꽃받침 등의 정보로 붓꽃의 품종을 알고 싶으면 관련 데이터 학습이 끝난 이 노드에게 물어보면 된다.

Test and Score 위젯

  • 데이터를 받아 학습하고 성능을 평가하는 위젯

AUC는 accuracy(정확도)로 0.987 = 98.7%의 정확도로 붓꽃의 정보를 보고 품종을 맞췄다는 뜻.


새로운 붓꽃 데이터 주고 맞춰보라 시키기

기존 붓꽃 데이터 형식으로 새로운 데이터 생성하기

  1. Transform 에서 “Create Instance” 위젯을 클릭한다.

  2. “Data Table” 노드와 “Create Instance” 노드를 연결한다.

  3. 연결하자 마자 Edit Links 창이 나타난다.

  4. “Clear All” 버튼을 누른다. 연결선이 없어진다.

  5. “Data“와 “Data“를 연결한다.

  6. OK 버튼을 누른다.

새로운 데이터 입력

들판을 다니다 붓꽃을 발견했다. 이 붓꽃의 품종을 확인하기 위해 꽃잎의 길이와 너비, 꽃받침의 길이와 너비를 측정했다. 그리고 이전에 학습시켜 놓은 붓꽃 AI 모델에게 그 값을 주면서 품종이 무엇인지 물어보는 과정이다.

하지만 실제로는 지금 우리 손에 붓꽃이 없기 때문에 기존 데이터를 참조해서 대략적인 값을 만들어 보겠다.

“Data Table” 노드를 더블클릭하면 위와 같이 3가지 품종의 데이터들을 볼 수 있다.

여기서 우리는 세토사의 기존 값을 참조해 새로운 값을 만들어 보겠다.

  1. 슬라이더를 조정해 꽃받침 길이를 4.9 와 비슷한 값으로 바꾼다.

  2. 슬라이더를 조정해 꽃받침 너비를 3.1 과 비슷한 값으로 바꾼다.

  3. 슬라이더를 조정해 꽃잎 길이를 1.6 과 비슷한 값으로 바꾼다.

  4. 슬라이더를 조정해 꽃잎 너비를 0.1 과 비슷한 값으로 바꾼다.

  5. 품종은 “?“로 바꾼다. 우리는 AI에게 이게 뭔지 물어보는 중이다. 학습할 때처럼 정답은 주지 않는다.

  6. “Append this instance to input data“의 체크를 푼다. 이 체크를 하면 기존 150개의 데이터에 지금 방금 내가 새로 입력한 데이터를 추가로 해서 151개의 데이터가 AI에게 넘어가게 된다. 우리가 원하는 건 지금 방금 발견한 붓꽃 하나의 품종이니 이 체크를 풀어 이 데이터 하나만 AI에게 넘어가도록 한다.

학습 결과가 바로 나타난다.

Linear Regression 결과창 지표 설명

보통은 R2와 RMSE 지표를 우선 본다.

MSE

  • Mean Squared Error. 평균 제곱 오차
  • 오차를 제곱해서 평균 낸 것.
  • 해석: 오차를 제곱했기 때문에 오차가 날수록 이 값은 매우 커진다. 오차를 크게 놓고 보고 싶을 때 사용. 단, 제곱된 상태라 금액에 대한 직관적인 이해는 어렵다.
  • 작을수록 학습이 잘 된 것 = 모델 성능이 좋은 것

RMSE

  • Root Mean Squared Error. 평균 제곱근 오차
  • MSE에 루트를 씌워 단위를 원래 주택가격 단위로 맞춘 것. 그래서 금액에 대한 직관적인 이해가 쉽다.
  • 해석: MSE의 장점인 큰 오차에 민감함은 유지하면서도 MAE처럼 실제 금액 단위로 해석 가능하다.
  • 작을수록 학습이 잘 된 것 = 모델 성능이 좋은 것

MAE

  • Mean Absolute Error. 평균 절대 오차
  • 실제 가격과 예측 가격의 에 절대값을 씌워 평균한 것.
  • 해석: 만약 MAE가 3이라면, 이 주택가격의 단위가 $1,000 이므로, 약 $3,000 정도 싸거나 비싸게 맞추는구나 라고 직관적으로 이해한다.
  • 작을수록 학습이 잘 된 것 = 모델 성능이 좋은 것

MAPE

  • Mean Absolute Percentage Error, 평균 절대 백분율 오차
  • 오차가 실제 가격의 몇 %수준인지를 비율로 나타낸 것.
  • 해석: 만약 MAPE가 15%라면, 실제 집값 대비 평균 15% 정도의 오차율을 보이며 예측한다.
  • 작을수록 학습이 잘 된 것 = 모델 성능이 좋은 것

sMAPE

  • Symmetric Mean Absolute Percentage Error, 대칭 평균 절대 백분율 오차
  • MAPE의 단점인 실제값이 너무 작으면 왜곡되는 문제를 보완하기 위해 분모에 실제값과 예측값의 평균을 넣어 대칭적으로 계산한 오차 비율
  • 해석: 만약 MAPE가 15%라면, 실제 집값 대비 평균 15% 정도의 오차율을 보이며 예측한다.
  • 작을수록 학습이 잘 된 것 = 모델 성능이 좋은 것

R2

  • R Squared. 결정계수. R2는 R의 제곱이라는 뜻.
  • 이 모델이 얼마나 많은 부분을 예측할 수 있는가를 나타내는 비율
  • 해석: 0~1의 값을 가지는데 이 값이 1이면 100% 완벽하게 집값을 맞추었다는 뜻.

K(계산에 사용할 이웃의 수) 변경

지금까지는 기본값인 K=5인 상태에서의 테스트였다.

K=100으로 바꾸고 테스트해 보자.


아까와 같이 x=0.112인 경우

똑같이 x=0.112로 주었는데,

  • 아까 k = 5일 때는 0.203581,

  • 지금 k = 100일 때는 0.266492 이다.

이유는,

X값이 0.2 ~ 0.4까지의 Y값은 모두 0.4이기 때문에 평균이 약간 올라갔다.

  1. 우리가 만든 “금리”, “측량횟수” 필드 중 먼저 “금리” 필드의 값을 조정하기 위해 “금리“를 입력한다.

  2. “금리” 필드의 0 ~ 1 사이의 값을 1 ~ 3 으로 조정하기 위해, “금리 * 2.0 + 1.0” 을 입력한다.

  3. 금리를 조정하는 수식이 완성된 것을 확인한다.

금리 := 금리 * 2.0 + 1.0


“측량횟수” 필드 값 조정

  1. “New” 버튼을 누르고 “Numeric“을 선택한다.

  2. “측량횟수” 필드의 값을 조정하기 위해 “측량횟수“를 입력한다.

  3. “측량횟수” 필드의 0 ~ 1 사이의 값을 100 ~ 500 으로 조정하기 위해, “측량횟수 * 400 + 100” 을 입력한다.

  4. 측량횟수를 조정하는 수식이 완성된 것을 확인한다.

  5. “Send” 버튼을 누른다.

  6. 닫기 버튼을 누른다.

측량횟수 := 측량횟수 * 400 + 100


조정된 데이터 확인

  1. Data 에서 “Data Table” 위젯을 추가한다. 그리고 “Formula” 노드와 “Data Table” 노드를 연결한다. 그리고 “Data Table” 노드를 더블클릭한다.

  2. Data Table 창이 나타나고 기존에 0~1 사이에만 있던 값들이 금리는 1.0 ~ 3.0, 측량횟수는 100 ~ 500 사이인 것을 확인할 수 있다.


feature와 target 정리

  1. Transform에서 “Select Columns” 위젯을 추가한다. “Data Table” 위젯과 연결한다. “Select Columns” 위젯을 더블클릭한다.

  2. Feature에 있던 “측량횟수“를 마우스로 끌어내려 Target에 붙인다.

이제 금리는 feature = 독립변수가 되었고,
측량횟수는 target = 종속변수가 되었다.


데이터 시각화

  1. Visualize 에서 “Scatter Plot” 위젯을 추가한다. “Select Columns” 노드와 연결한다. “Scatter Plot” 노드를 더블클릭한다.

  2. Scatter Plot를 보며 대략적인 금리와 측량횟수 간의 관계를 확인한다.

확인이 끝나면 Scatter Plot는 삭제한다.


Decision Tree 모델 추가

  1. Model 에서 “Tree” 위젯을 추가한다. 그리고 “Select Columns” 노드와 연결한다.

Decision Tree 모델 성능 테스트

  1. Evaluate 에서 “Test and Score” 위젯을 추가한다.

  2. “Tree” 노드와 “Test and Score” 위젯을 연결한다. 이 때 선에 “Learner” 라고 적혀 있어야 한다.

  3. “Select Columns” 노드와 “Test and Score” 위젯을 연결한다. 이 때 선에 “Data” 라고 적혀 있어야 한다.

  4. “Test and Score” 위젯을 더블클릭한 다음, “RMSE” 항목을 확인한다.

  5. Test and Score 창에서 “R2” 항목을 확인한다.


Decision Tree 성능 조정

  1. “Tree” 노드를 더블클릭한다.

  2. “Tree” 노드의 성능을 조정하기 위한 값의 기본값이 다음과 같았음을 확인한다.

  • “Limit the maximal tree depth to” : 5

위와 같은 설정일 때 R2의 값이 0.994였음을 알 수 있다.


“Tree” 모델의 Depth 줄이기

Decesion Tree 모델의 depth는 이 모델이 몇 단계나 깊이 들어가며 계속 가지를 쳐 나갈건지에 대한 옵션값이다.

Depth를 줄이면 세밀하게 정리하는 단계가 줄어들어 정확도가 떨어진다.

R2의 값이 depth가 기본값인 5일때는 0.994였는데, 2로 줄이니 0.943으로 떨어진 것을 확인할 수 있다.


“Tree” 모델의 Depth 늘리기

depth를 늘리면 더 많은 가지를 치며 세밀하게 정리해 정확도가 높아지지만 너무 높이면 과적합이 될 수 있다.

depth를 10으로 올리니 R2의 값이 0.996으로 depth=5일 때(0.994)보다 높아진 것을 볼 수 있다.

실습 9 - Gradient Boosting

Random Forest 는 실습 8. Random Forest 실습을 기반으로 합니다. 먼저 이 실습을 먼저 완성해 주세요.

실습 8 완성 모습


Gradient Boosting 모델 추가

  1. Model에서 “Gradient Boosting” 위젯을 추가한다.

  2. “Gradient Boosting” 노드와 “Test and Score” 위젯을 연결한다. 이 때 선에 “Learner” 라고 적혀 있어야 한다.

  3. “Select Columns” 노드와 “Gradient Boosting” 위젯을 연결한다. 이 때 선에 “Data” 라고 적혀 있어야 한다.


Gradient Boosting 모델 성능 테스트

  1. “Test and Score” 노드를 더블클릭한다.

Test and Score 창이 나타나며 결과를 보여준다.

같이 연결된 기존 Decision Tree, Random Forest 모델과의 결과도 비교해 볼 수 있다.

데이터가 매우 예측가능한 선형 구조여서 각 모델 간의 차이가 크지 않지만, 이론상으로는 Decision Tree < Random Forest < Gradient Boosting 순으로 정확도가 높아진다. (오른쪽으로 갈 수록 정확도가 높음)

Gradient Boosting 모델 성능 설정

  1. “Gradient Boosting” 노드를 더블클릭한다.

  2. 사용할 Decision Tree의 수의 기본값이 100이다.

  3. Learning rate의 기본값은 0.1이다. 이 값이 낮으면 이전 나무의 오차를 강하게 반영하지만 속도가 느려지고, 이 값이 높으면 빠르게 학습하지만 이전의 오차를 지나쳐 버릴 수 있는 문제가 생긴다. 이 값은 보통 0.01 ~ 0.3 정도의 값을 가진다.

Learning rate의 값을 낮추면 Decision Tree의 수는 늘려야 한다. 꼼꼼하게 학습하려면 Decision Tree의 수가 많아야 한다.
